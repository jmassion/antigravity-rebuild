📋 Things to Know

What's Working
The demo shows 6 browser surfaces loaded inside iframes that you can mouse over, click, and scroll. Each has an editable URL bar — type any URL and press Enter to navigate. The WebSocket protocol (rooms, presence, capability tokens, locks) is fully operational underneath.
Iframe Limitations
Many major websites (Gmail, GitHub, Notion, Figma) block iframe embedding via X-Frame-Options or CSP headers. There's no browser-side workaround for this — it's a security policy set by those websites. The current demo uses sites that DO allow embedding: Wikipedia, YouTube (embed URL), OpenStreetMap, Internet Archive, CodePen, and Google Maps.

To show blocked sites like Gmail or GitHub, you'd need a server-side proxy that strips the X-Frame-Options headers — or a Chrome extension that overrides them. The real TabSpace architecture solves this by using CDP (Chrome DevTools Protocol) pixel streaming instead of iframes, which is the "next step" in the README.
Architecture Under the Hood
Even though the demo looks simple, the full multiplayer protocol is running: Server (WebSocket room authority + HMAC capability signing), Host (publishes surfaces into rooms), Client (renders surfaces, receives caps). All 3 run together. Multiple browser tabs can join the same room with different user IDs and see each other's presence.
How to Test Multi-User
Open http://localhost:8787 in a second browser tab. The default user is user-justin. To change it, edit the hidden input in index.html or add ?user=user-susannah to the URL (I can add URL param support if you like).
