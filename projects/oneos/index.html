Build Methodology

Generative × Procedural

The vision is generative. The path is procedural. Neither works alone.

🌀 Generative (Vision)

Start with the end feeling.

  • This page IS the vision
  • Demos show end state before code exists
  • Language spec emerges from desired UX
  • Each prototype is a miniature universe

📋 Procedural (Path)

Break into executable steps.

  • Wiki holds PRD + phase breakdowns
  • Every task ships a working artifact
  • Kanban tracks dependencies
  • Playgrounds test concepts in isolation

✦ The Merge: Emergent Building

Every procedural step validated against the generative vision. Every insight decomposed into tasks. The system converges because both forces point at the same truth.
