Under the Hood

How It Works

A three-tier architecture designed for real-time multiplayer browsing at scale.

1

WebSocket Room Server

Authoritative Node.js server managing rooms, presence, capability tokens, epoch-based revocation, and binary frame forwarding. Every interaction is permission-gated.

2

CDP Host / Browser Engine

Playwright-powered headless Chrome instances that capture screencast frames via CDP and stream JPEG pixels over WebSocket. Mouse & keyboard input is injected back into the real browser.

3

Spatial Web Client

Interactive frontend with iframe-first viewports, CDP canvas overlays, 8 view modes, and a Three.js 3D sphere renderer. Auto-connects to the room server for live surface data.

4

Future: Agent Layer

AI agents receive capability tokens to interact with browser surfaces — spawning pages, navigating, extracting data, and executing multi-step workflows autonomously.
