What is Vercel MCP?

The Model Context Protocol (MCP) is a standard interface that lets AI assistants communicate with external tools. Vercel's MCP server at https://mcp.vercel.com gives AI tools like Gemini Code Assist secure, OAuth-authenticated access to manage your Vercel projects — listing deployments, checking logs, deploying code, and managing domains.


Think of it as a USB-C port for AI: one connection, many capabilities. Instead of manually clicking through dashboards, your AI assistant can create projects, trigger deployments, and check status directly.

💡 How CNAME Records Work When you add a custom domain to a Vercel project, Vercel gives you a CNAME target: cname.vercel-dns.com. You then add a CNAME record in SiteGround's DNS Zone Editor pointing your subdomain to this target. Vercel handles SSL automatically.

Gemini Code Assist Setup

Configuration File

Add this to ~/.gemini/settings.json (already configured):

{
  "mcpServers": {
    "vercel": {
      "command": "npx",
      "args": ["mcp-remote", "https://mcp.vercel.com"]
    }
  }
}

After adding, restart Gemini Code Assist. It will prompt you to authenticate with Vercel via OAuth on first use.

SiteGround DNS Configuration

⚠️ Manual Step Required SiteGround does not have a public API for DNS management. CNAME records must be added through the Site Tools UI. The Deployment Manager dashboard generates exact copy-paste values to make this fast.

Step-by-Step: Add a CNAME Record

  1. Log in to SiteGround Client Area → click Websites → select holodeckos.com → click Site Tools
  2. Navigate to Domain → DNS Zone Editor
  3. In the record type dropdown, select CNAME
  4. Enter the subdomain prefix in the Name field (e.g., noahos for noahos.holodeckos.com)
  5. In Resolves to, paste: cname.vercel-dns.com
  6. Set TTL to 3600 (or leave default)
  7. Click Create — DNS propagation takes 5-30 minutes
